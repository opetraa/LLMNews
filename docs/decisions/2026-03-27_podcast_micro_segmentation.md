# 20 Micro-Segmentation for 1-Hour Podcast Generation

## 상태: 제안됨
## 날짜: 2026-03-27

## 맥락
기존 아키텍처는 5개의 광범위한 세그먼트(Foundation Models, Papers 등) 단위로 Vertex AI를 호출하여 60분 분량의 팟캐스트를 생성하려 했습니다. 그러나 LLM은 단일 호출 시 핵심만 요약하려는 경향(Token compression)이 있어 각 챕터가 2~3분으로 축소, 전체 길이가 15~20분에 그치는 한계가 발생했습니다. 이를 해결하기 위해 프롬프트를 억지로 늘리는(Hallucination 유발) 대신, 세그먼트를 20개로 잘게 쪼개는 아키텍처 변경을 제안합니다.

## SOTA 리서치 및 비교 매트릭스 (3-Layer Search)

| 관점 | NotebookLM (Google) | Wondercraft AI (Industry) | **Tractara Podcast (제안)** |
|---|---|---|---|
| **오디오 길이 제어** | 사용자가 입력한 소스 양에 의존적 (제어 어려움) | 타임라인 기반 수동 편집으로 2~4분 단위 블록 시퀀싱 | **사전 정의된 20개 마이크로 세그먼트** 배열로 자동화된 50~60분 타임라인 보장 |
| **인지 부하 (Cognitive Load)** | 대화형 흐름에 묻어감 (때로 정보 과부하) | 유저가 직접 챕터링 | 하나의 주제당 **2.5분(최적 인지 시간)** 제한으로 청취자 피로도 최소화 |
| **SOTA vs Tractara 정렬** | 자동화된 자연스러운 합성 | 강력한 프로덕션 제어 | **완전 자동화(NotebookLM) + 정밀한 분량 제어(Wondercraft)** 의 결합 |

## 결정
팟캐스트의 대본 구조를 5개의 대분류에서 **20개의 마이크로 세그먼트(세부 주제)**로 변경합니다. (예: 주요 기업 실적, 오픈소스 LLM 동향, RAG 논문 리뷰, 빅테크 루머 등)

## 대안
* **단일 프롬프트 분량 강제 (기각)**: "최소 2000단어 이상 작성"과 같이 지시할 수 있으나, LLM 특성상 불필요한 반복어(Filler words)나 환각(Hallucination)이 심해져 오디오 품질을 크게 저하시킵니다.

## 결과
* **긍정적 효과**: 억지스러운 텍스트 늘리기 없이, 각 주제별로 가장 영양가 있는 2~3분짜리 대본이 20개 모여 **자연스러운 고품질 50분 팟캐스트**가 완성됩니다.
* **부정적 효과(비용)**: AI 호출 횟수가 5회에서 20회로 증가하여 약간의 API 토큰 소모량 증가가 예상되나(Gemini 1.5 Flash 기준 미미한 수준), 현재 워크플로우 상의 I/O 병목은 크지 않을 것으로 판단됩니다.
